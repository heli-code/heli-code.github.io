# Vibe Music 项目启动说明

## 项目概述
这是一个完整的音乐播放系统，包含以下组件：
- **后端服务**: Java Spring Boot (端口 8080)
- **前端客户端**: Vue.js (端口 5173) 
- **管理后台**: Vue.js (端口 5174)
- **依赖服务**: Redis、MySQL、MinIO

## 快速启动
运行根目录下的 `start_services.bat` 文件即可自动启动所有服务。

## 手动启动步骤

### 1. 启动依赖服务

#### Redis
- 确保Redis服务已安装并运行
- 默认端口: 6379
- 启动命令: `redis-server` 或通过Windows服务启动

#### MySQL  
- 确保MySQL服务已安装并运行
- 默认端口: 3306
- 启动命令: `net start MySQL80`
- 数据库名: vibe_music
- 导入SQL文件: `vibe-music-server/sql/vibe_music.sql`

#### MinIO
- 下载MinIO: https://dl.min.io/server/minio/release/windows-amd64/minio.exe
- 将minio.exe放入 `minio/` 目录
- 启动命令: `minio.exe server data --console-address 127.0.0.1:9090 --address 127.0.0.1:9000`
- 或使用Docker: `docker run -p 9000:9000 -p 9090:9090 minio/minio server /data --console-address ":9090"`

### 2. 启动后端服务
```bash
cd vibe-music-server
mvn clean package -DskipTests
java -jar target/vibe-music-server-0.0.1-SNAPSHOT.jar
```

### 3. 启动前端服务

#### 前端客户端
```bash
cd vibe-music-client
pnpm install  # 或 npm install
pnpm dev      # 或 npm run dev
```

#### 管理后台
```bash
cd vibe-music-admin
pnpm install  # 或 npm install
pnpm dev      # 或 npm run dev
```

## 注意事项
1. 首次启动需要安装依赖，请耐心等待
2. 确保所有依赖服务正常运行后再启动前端和后端

4. 建议使用pnpm包管理器以获得更好的性能
